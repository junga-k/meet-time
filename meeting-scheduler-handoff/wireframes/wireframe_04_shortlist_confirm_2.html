구현 노트 (Claude Code 참고용)

  • 접근 대상: PARTICIPANT.role='선택'인 참가자, MEETING.stage='선택확인중'일 때만 접근 가능 — 그 전에는 회의 목록(화면 1)에 "대기 중"으로만 표시되고 클릭 비활성. 이 화면은 화면 3(슬롯 응답)과 헤더·탭 컴포넌트를 공유하되 배지만 "선택 참석자"로 바뀜(화면 3은 "필수 참석자")
  • 참석 형태 선택은 화면 3과 동일한 컴포넌트(대면만/온라인만/둘 다 가능, 회의당 1회, PARTICIPANT.attendance_mode) — 선택 전에는 후보 카드 목록 위에 잠금 오버레이가 뜨고 "응답 완료" 버튼도 비활성화됨(화면 3과 동일한 게이팅 패턴)
  • 회의 형태(MEETING.mode) 참고 표시 — 이미 필수 참석자 응답 기준으로 자동 계산되어 확정된 값이며, 이 화면에서 변경 불가(읽기 전용). "참고용"이라는 점을 우측에 작게 병기해 오해 방지
  • 후보 목록은 TIME_SLOT.is_shortlisted=true인 5개만 표시(전체 그리드 아님) — 화면 3의 30분 단위 전체 그리드와 달리 가벼운 확인만 목적. 각 카드는 날짜·시간을 표시하고 가능/불가 2択만 제공(기피 없음 — 세밀한 선호가 아니라 참석 가능 여부만 필요하기 때문)
  • 카드 기본값은 가능(화면 3과 동일하게 SLOT_RESPONSE 레코드가 없는 경우의 기본값 규칙을 따름) — 참석이 어려운 후보만 "불가"로 클릭해서 바꾸면 됨. 화면 상단 안내 문구에서는 "표시하지 않으면 가능 처리"라는 부연 설명을 생략함(요청에 따라 제거 — 대신 확인 팝업에서 실제로 가능 처리될 후보를 구체적으로 보여주므로 중복 설명 불필요)
  • "응답 완료" 버튼: 참석 형태 선택 전엔 비활성화. 클릭 시 확인 팝업이 뜨고 확인해야 PARTICIPANT.responded_at 기록. 선택 참석자 전원이 완료하면 optional_response_deadline 전이라도 즉시 MEETING.stage='확정'으로 전환(조기 진행). 확인 팝업 내용은 "표시하지 않은 후보는 가능 처리"라는 일반 문구 대신, 그 시점에 "가능"으로 표시된 후보들의 실제 날짜·시간을 나열해서 보여줌(예: "가능: 목 7/10 14:00~15:00, 금 7/11 10:30~11:30, ...") — 사용자가 확정 직전에 무엇이 "가능"으로 제출되는지 구체적으로 확인할 수 있도록 함. 5개 전부 불가인 경우는 별도 분기(위 항목)로 처리되어 이 목록이 뜨지 않음
  • 5개 전부 "불가"로 응답한 경우 분기 처리: 완화요청·재조율을 트리거하지 않고 confirmation_status='불참'으로만 기록, 나머지 진행(필수 참석자 기준 확정)은 그대로 진행됨. 확인 팝업 문구도 이 경우 다르게 표시(불참 처리됨을 명확히 안내)하고, 확인 직후 이 화면에서는 "불참으로 기록됐어요"라는 1차 안내만 표시함(이 와이어프레임은 1인 시점 데모).
  • 불참 처리 이후의 2차 안내(이 화면 밖, 별도 시점): 위 1차 안내가 전부가 아님 — MEETING.stage가 '확정'으로 전환되는 시점에 아직 confirmation_status='불참'으로 남아있는 선택 참석자에게 NOTIFICATION(type: 불참안내)이 별도로 발송됨. 회의 시작 전이면 "참석으로 변경할 수 있다"는 기회를 함께 안내(화면 9 회의상세의 "참석으로 변경" 버튼으로 연결), 회의 시작 후라면 "회의록을 확인해주세요"만 안내(v2 회의록 완성 전까지는 문구만, 실제 이동 링크 없음). 상세 규칙은 screen_specs.md 화면 12 "선택 참석자 불참 처리 후속 안내" 절 참고 — 이 화면은 그 흐름의 첫 단계만 보여주는 것이며, 전체 라이프사이클은 별도 화면(9·11)에 걸쳐 있음
  • 확정 안내 배너("확정 후엔 일정 변경이 어려워요 · 제출된 응답은 시스템이 자동으로 확인해 조율해요")는 화면 3과 동일하게 "응답 완료" 버튼 바로 위(footer)에 위치 — 위쪽에 참석형태·모드정보·안내·개인정보 문구가 쌓여 카드 목록 영역이 눌리는 문제를 방지하기 위해 처음부터 이 위치로 설계함(화면 3에서 겪었던 레이아웃 겹침 이슈를 사전에 반영)
  • 반영 완료: 배너 문구는 project_handoff.md "AI 활용 범위" 절(이 흐름은 전부 규칙 기반 자동화이며 LLM 불필요)과 일치하도록 "AI가 확인 후 조율해요" → "제출된 응답은 시스템이 자동으로 확인해 조율해요"로 수정 완료(화면 3과 동일한 이슈였음)
  • 개인정보 안내("응답 내용은 개별 공개되지 않고, 집계 결과만 전달돼요")도 화면 3과 동일한 이유로 추가 — screen_specs.md 화면 5·6 스펙 기준으로 이미 참인 내용을 사용자에게도 명시
