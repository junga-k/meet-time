구현 노트 (Claude Code 참고용)

  • 전통적인 아이디/비밀번호 로그인 폼이 아님 — 두 버튼 모두 OAuth2/OIDC 리다이렉트 방식. "사내 계정으로 로그인"은 회사의 사내 SSO(예: 사내 그룹웨어 IdP, Microsoft Entra ID/Azure AD, Okta 등)로, "Google로 로그인"은 Google OAuth(회사 도메인으로 제한 가능)로 각각 리다이렉트 후 콜백으로 돌아옴
  • 실제 배포 시 두 버튼을 동시에 노출하지 않는 것을 전제로 설계함 — 회사마다 실제로 쓰는 인증 수단이 하나뿐이라, 배포 조직의 설정값에 따라 둘 중 하나만 노출하는 구성으로 구현 권장 (와이어프레임에서는 시연을 위해 둘 다 노출)
  • 자체 회원가입 절차 없음 — 계정 자체는 SSO/Google 쪽에서 관리. 최초 로그인 시에만 USER 레코드를 새로 생성하고 "최초 프로필 설정" 화면(신규)으로 이동시켜 SSO가 주지 않는 필드(부서·직위·직책·내선번호·메신저ID)를 보완 입력받음. 이미 가입된 사용자는 바로 "내 회의" 목록(화면 1)으로 이동
  • 이 화면 자체가 스플래시를 겸함 — 앱 최초 진입 시 loading-overlay(로고+로딩 인디케이터)가 잠깐 표시된 뒤 로그인 여부에 따라 자동 분기(로그인 상태면 이 화면을 건너뛰고 바로 내 회의 목록으로, 아니면 이 화면 노출). 별도의 전용 스플래시 화면은 만들지 않기로 결정함(open_decisions_tracker.md §0 참고)
  • 이 와이어프레임은 단일 화면 목업이라 실제 리다이렉트 대신 토스트로 다음 이동 화면을 표시하는 것으로 대체 시연함 (데모 편의상 "사내 계정으로 로그인" 클릭 시 최초 사용자 흐름을, "Google로 로그인" 클릭 시 기존 사용자 흐름을 시연하도록 구분해둠 — 실제로는 두 버튼 다 동일한 최초/기존 분기 로직을 탄다)
