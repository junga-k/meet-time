구현 노트 (Claude Code 참고용)

  • 아이콘은 전부 인라인 SVG — 참석자 삭제(x)·조직도 chevron·추가(plus)/이미추가됨(check) 아이콘이 CDN 아이콘 웹폰트(<i class="ti ti-...">)에 의존하고 있었는데, 네트워크가 막힌 환경에서 안 보이는 문제가 있어 전부 인라인 SVG로 교체(상단에 미리 정의된 ICONS 객체 재사용). wireframe_01_slot_response.html·wireframe_02_agenda.html과 동일한 방식
  • 회의 후보 기간은 캘린더에서 시작일·종료일을 직접 선택. 오늘로부터 최소 3영업일 이후 날짜만 선택 가능(그 이전은 회색으로 비활성). 필수·선택 응답 마감은 이 선택에 따라 자동 역산되어 미리보기에 표시됨 — 별도 입력 필드 없음
  • 온/오프라인 모드, 장소, 화상링크는 이 화면에서 입력받지 않음 — 모드는 필수 참석자들이 슬롯 응답 시 고르는 참석 형태(대면/온라인/무관)를 집계해 자동 계산되고, 장소·링크는 시간이 확정된 뒤 회의실 자동 매칭/화상링크 자동 생성으로 처리됨 (화면 3, 5 참고)
  • 저장 시 MEETING 레코드 생성 + 선택한 기간만큼 TIME_SLOT 30분 단위로 일괄 생성
  • 예상 회의 시간은 주최자가 시간·분을 직접 입력(숫자 입력 필드 2개 — 시간 0~8, 분 0~59). 기본값은 1시간 0분. 기존 스펙(screen_specs.md)엔 "기본 1시간 고정값"으로 적혀 있었으나 이번 수정으로 직접 입력 가능하게 바뀜 — 스펙 문서 업데이트 필요. MEETING에 duration_minutes 필드 추가가 필요함(현재 스키마엔 없음), 저장 시 이 값(시간×60+분) 기준으로 각 TIME_SLOT의 회의 길이가 정해짐(30분 단위 슬롯 자체의 간격과는 별개 개념). 두 입력 모두 0이면(빈 값 포함) 유효하지 않은 것으로 처리해 "회의 만들기" 버튼을 막음
  • 안건 마감은 회의 후보 기간과 동일한 스타일의 캘린더로 날짜를 선택(단, 범위가 아닌 단일 날짜 선택 — 오늘 이전 날짜는 비활성화)하고, 캘린더 바로 아래 시:분 드롭다운으로 마감 시각을 지정. 분 단위는 TIME_SLOT 생성 단위(30분)에 맞춰 00/30만 제공. 날짜 미선택 시 "회의 만들기" 버튼이 막히도록 검증에 추가함(기존엔 datetime-local 네이티브 입력이라 별도 검증이 없었음) — 저장 시 선택된 날짜+시간을 합쳐 MEETING.agenda_deadline(datetime)에 저장
  • 참석자는 두 가지 방식으로 찾음: ① 검색창에 이름을 치면 드롭다운으로 후보가 뜸(빠른 검색) ② 검색창 오른쪽 조직도 아이콘을 누르면 부서별 아코디언이 있는 전체화면 모달이 열림(부서 단위로 훑어볼 때). 부서가 많은 회사에서도 기본 폼 화면 길이가 늘어나지 않도록 조직도는 모달로 분리함
  • "추가된 참석자" 목록은 검색·조직도 영역과 분리된 별도 섹션 헤더("추가된 참석자 (N명)")로 구분 — 조직도 모달 안에서는 이미 추가된 사람에게 체크 아이콘이 떠서 중복 추가를 막음
  • 참석자를 추가하면 역할이 비어있는 상태(role: null)로 들어가고, 필수/선택 중 하나를 명시적으로 선택해야 함 — 선택 전에는 토글 테두리가 빨갛게 표시되고, 저장 시에도 미선택 참석자가 있으면 막힘
  • 주최자 본인은 자동으로 PARTICIPANT(역할: 필수)로 추가되며 목록에서 제거 불가
  • 참석자 조직도는 사내 USER 디렉터리를 부서별로 그룹핑한 것 (이 와이어프레임은 데모용 7명 목업, 5개 부서)
  • 검증 규칙: 제목 필수, 참석자(주최자 외) 최소 1명, 전원 필수/선택 역할 선택 완료, 캘린더에서 시작일·종료일 모두 선택, 예상 회의 시간이 0보다 커야 함(시간·분 직접 입력값 합산), 안건 마감 날짜 선택. 주최자는 항상 자동으로 필수 참석자에 포함되므로 실질적 검증은 "참석자 1명 이상 + 역할 전원 선택 + 후보 기간 선택 + 예상 회의 시간 입력 + 안건 마감 날짜 선택"
  • "회의 만들기" 버튼은 위 검증 규칙을 모두 충족하기 전까지 비활성화(회색, 클릭 불가). 조건이 남아있으면 버튼 아래 안내 문구로 무엇이 더 필요한지 표시
  • "회의 만들기" 클릭 시 확인 팝업(제목·참석자 수·후보 기간 요약)이 한 번 더 뜨고, "확인"을 눌러야 실제로 저장됨 (모든 화면 공통 규칙 — screen_specs.md 참고)
  • 저장 성공 시 필수 참석자 전원에게 초대 NOTIFICATION 즉시 발송, 선택 참석자에게는 "대기 중" 알림만 발송 + 회의 대시보드(화면 5)로 이동
  • 상단 헤더는 다른 화면과 달리 탭·아바타 없음 — 아직 존재하지 않는 회의라 "내 정보" 컨텍스트가 필요 없음
