구현 노트 (Claude Code 참고용)

  • 안건 카드 = AGENDA 레코드 1개 (id, meeting_id, title, description, is_required, attachment_url)
  • 정렬 규칙: is_required=true 우선, 그다음 등록순 (오래된 순)
  • 안건 마감(agenda_deadline)이 지나도 등록 폼은 계속 열려있음 — 마감은 회의 진행 여부에 영향 없고, 주최자에게 리마인드만 발송(별도 배치 작업)
  • 첨부파일 버튼은 목업 — 실제 구현에서는 파일 업로드 후 attachment_url에 저장된 경로 반영. 파일이 첨부된 상태에서는 파일명+삭제(x) 버튼이 위, "파일 선택"(다른 파일 선택) 버튼이 그 아래로 배치되어, 다른 파일로 교체하거나 완전히 제거하는 두 동작을 구분함
  • "안건 등록"/"수정 완료" 버튼은 안건 제목이 입력되기 전까지 비활성화 상태(회색, 클릭 불가). 설명·필독·첨부파일은 선택 입력이라 버튼 활성화 조건에 포함 안 됨
  • 카드 우측 상단 "⋯"(수정/삭제 메뉴)는 작성자 본인에게만 노출 — AGENDA.author_id === 현재 로그인 user_id일 때만 렌더링. 서버에서도 동일 조건으로 권한 검증 필요 (프론트 숨김만으로는 보안 불충분)
  • 상단 헤더·탭 구조는 화면 3(슬롯 응답)과 동일한 컴포넌트 재사용
  • 카드에 작성자 이니셜 아바타 + 이름 표시. USER.profile_image_url이 있으면 사진, 없으면 이름 첫 글자를 원형 배지에 표시 (예: "박서연" → "박"). 새 안건 등록 시 작성자는 현재 로그인한 사용자로 자동 설정, 별도 선택 UI 없음
  • 이모지 대신 Tabler Icons 라인 스타일 사용 (앱 전체 공통 규칙) — 첨부파일·수정삭제 메뉴 아이콘도 동일 세트로 통일. 이전 버전은 CDN 아이콘 웹폰트(<i class="ti ti-...">)에 의존해 네트워크가 막힌 환경에서 아이콘이 안 보이는 문제가 있었음 — wireframe_01_slot_response.html·wireframe_03_meeting_create.html과 동일하게 인라인 SVG로 전부 교체(상태바·뒤로가기·첨부클립·삭제·탭바·케밥메뉴 포함)해 외부 리소스 없이도 항상 렌더링되도록 함. 사람 아바타(이니셜/사진)는 별도 컴포넌트로 유지
  • 안건을 등록·수정·삭제하면 작성자 본인을 제외한 나머지 참가자에게 NOTIFICATION 레코드가 생성되고, 그들의 탭바 "알림" 아이콘에 빨간 점이 뜸. 이 와이어프레임은 1인 시점 데모라 본인 화면에서 직접 확인은 불가 — 배지는 항상 켜둔 상태로 표시
  • "새 안건 추가" 폼은 상시 노출 대신 목록 아래 "안건 추가" 버튼 → 하단에서 위로 슬라이드업되는 바텀시트형 팝업으로 전환. 화면 전체를 덮는 풀패널이 아니라, 이전 인라인 폼과 비슷한 크기의 박스가 탭바 바로 위쪽에서 올라오고 뒤로 목록이 어둡게 비치는 구조(딤 배경 페이드인 + 박스 슬라이드업 애니메이션, transform: translateY 기반). 목록 화면을 더 간결하게 보여주기 위한 변경이며, 폼 필드·유효성 검증(제목 필수) 로직은 기존과 동일하게 팝업 안에서 그대로 동작. "수정" 메뉴 클릭 시에도 같은 팝업이 해당 안건 데이터로 채워진 채 열림. 등록/수정 완료, X 버튼 클릭 시 팝업 닫힘
