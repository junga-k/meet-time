  • "소속"(부서·직위·직책)도 항상 읽기 전용 — 최초 프로필 설정(화면14)과 동일하게 회사 디렉토리(HRIS/SSO 그룹 속성)에서 자동 동기화되는 정보로 최종 확정(트래커 §2-9, §3-7 해결). 수정 모드로 전환해도 "소속" 섹션은 입력 필드로 바뀌지 않고 계속 읽기 전용 뷰 그대로 노출됨 — 값이 틀리면 사내 HR/IT 시스템에서 갱신해야 함. 수정 가능한 항목은 "연락처"(휴대폰·내선번호·메신저ID)뿐
  • "직급" 표기를 전부 "직위"로 변경(표시 텍스트만 변경, 내부 필드/변수명은 기존 rank 그대로 유지)
  • 수정 모드의 "연락처" 입력 필드에 placeholder 대신 별도 라벨(field-label)을 붙임 — 화면13(최초 프로필 설정)과 동일한 패턴 및 텍스트 색상 위계(섹션 제목은 진한 색, 필드 라벨은 옅은 색) 적용
  • 휴대폰번호·내선번호 입력 시 숫자만 입력해도 자동으로 하이픈 포맷(예: 010-1234-5678) 적용 — 화면13과 동일한 formatPhoneNumber() 로직 재사용
  • 상단 "수정" 링크 클릭 시 읽기 전용 뷰 → 입력 필드로 전환, 저장 버튼이 있는 하단 푸터가 함께 나타남. 저장 후에는 다시 읽기 전용 뷰로 복귀
  • 값이 없는 필드(직책·내선번호 등 nullable)는 읽기 전용 뷰에서 "-"로 표시
