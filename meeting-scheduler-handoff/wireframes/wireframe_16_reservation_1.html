구현 노트 (Claude Code 참고용)

  • 기존 스펙엔 없던 완전 신규 화면 — 탭바 1번째 탭("예약")에 대응하는 화면이 없던 것을 보완. 두 개의 서브탭으로 구성: "회의 예약"과 "회의실 현황"
  • "회의 예약" 서브탭은 새 폼을 만드는 게 아니라 화면 2(회의 생성, wireframe_03_meeting_create_1.html)의 진입점 — 이 화면에서는 콜투액션 카드 + 최근 만든 회의 목록만 보여주고, "회의 만들기 시작" 클릭 시 기존 화면 2로 이동. 회의 목록(화면 1) 하단의 "새 회의 만들기" 버튼과 동일한 목적지라, 진입 경로만 하나 더 생긴 것("탭바 예약" 경유 vs "내 회의 목록 하단 버튼" 경유)
  • "최근 만든 회의" 목록은 이 화면 전용 별도 데이터가 아니라 MEETING에서 organizer_id=나인 것만 created_at 내림차순(최신 생성순)으로 몇 건 보여주는 것(화면 1 "내 회의" 목록과 데이터 소스는 같고 필터·정렬만 다름 — 화면 1은 마감 임박순, 여기는 생성일순). "응답과 헷갈리지 않도록" 라벨을 "최근 예약한 회의"에서 "최근 만든 회의"로 변경
  • 목록 항목 클릭 시 진행 단계에 맞는 화면으로 이동(화면 1의 상태·역할별 분기 규칙과 동일한 원칙, 주최자 관점으로 단순화) — 제안중(응답 수집 중) → 회의 대시보드(화면 5, 주최자 전용) / 확정 → 회의 상세(화면 9). 데모에서는 data-stage 값에 따라 토스트로 이동을 시뮬레이션
  • "확정됨" → "확정"으로 표현 통일(다른 화면 상태 배지 표기와 맞춤)
  • CTA 카드 본문에서 "모드" → "회의 형태"로 용어 통일(§2-19 결정과 동일하게)
  • "회의실 현황" 서브탭이 진짜 신규 기능 — ROOM 전체 목록과 각 회의실의 예약 현황(ROOM_BOOKING)을 타임라인으로 표시. 이 화면은 조회 전용이며 회의실을 직접 예약하는 액션은 없음(회의실 예약은 항상 회의 확정 시점에 시스템이 자동으로 처리 — db_model_mvp.md "회의실 자동 예약" 규칙 참고)
  • 날짜·시간 검색 추가 — 상단 날짜·시간 입력 후 "조회" 클릭 시 해당 시점 기준으로 각 회의실의 예약 여부를 다시 계산해 배지에 반영. 이 와이어프레임은 정적 목업 데이터라 시간대만 재계산해서 시연 — 실제 구현 시엔 선택한 날짜의 ROOM_BOOKING을 서버에서 다시 조회해야 함
  • 타임라인 막대는 시간 텍스트 없이 색상으로만 표시하고, 배경 눈금선은 09~20시 매시간 동일 간격으로 표시(TICK_HOURS, 12개 선)해 정확한 위치를 가늠할 수 있게 함 — 다만 그 아래 시간 텍스트 라벨은 매시간 다 적으면 서로 겹쳐서 2시간 간격(LABEL_HOURS: 09·11·13·15·17·19)만 표기. 같은 회의실에 예약이 여러 건 붙어 있어도 구분되도록 5색 고정 팔레트를 순서대로 배정(자유/예약 배지의 초록·빨강과는 무관한 별도 색상 — 배지 색은 계속 "지금 비어있음/예약중" 의미로만 사용)
  • 회의실 카드 클릭 시 회의실 상세정보(위치·수용 인원·보유 장비·특이사항) → (16px 간격) → 오늘 예약 현황(시작~종료 시간, 주최자, 회의 제목) 순으로 보여주는 전체화면 패널이 열림 — 화면 2(회의 생성)의 "부서 조직도" 모달과 동일한 풀스크린 오버레이 패턴 재사용. 예약이 없는 회의실은 "예약된 일정이 없어요" 안내 표시. 예약 목록 각 행은 "주최자 : {직위} {이름} (총 참석인원 : N명)" 형식으로 표기(예: "주최자 : 대리 김정아 (총 참석인원 : 6명)")
  • 날짜 검색 UI를 네이티브 <input type="date">에서 화면 2(회의 생성)와 동일한 커스텀 캘린더 컴포넌트로 교체 — 브라우저 기본 날짜 선택기는 CSS로 스타일링이 불가능해 디자인 시스템과 어긋났음(이전 버전 스크린샷에서 확인된 문제)
  • 캘린더 레이아웃 조정 — 월 이동 화살표를 "YYYY년 M월" 라벨에 바짝 붙여 하나의 묶음(calendar-month-nav)으로 배치하고, 그 묶음과 같은 줄 오른쪽 끝에 "오늘"(현재 날짜로 바로 이동 + 패널 닫힘) 버튼을 추가 배치. 헤더는 grid-template-columns: 1fr auto 1fr 구조로 좌측에 보이지 않는 스페이서를 둬서 화살표+라벨 묶음이 "오늘" 버튼 유무와 무관하게 줄 전체 기준 정가운데에 오도록 정렬(기존엔 space-between이라 묶음이 왼쪽으로 치우쳐 보였음). 하단 버튼은 "삭제"(선택 해제, 옅은 테두리)와 "선택"(현재 선택된 날짜를 그대로 확정하고 패널만 닫음, 진한 배경 채움)으로 구성 — 기존엔 하단에 "오늘"이 있었지만 그 기능이 상단으로 옮겨가면서 "선택"(확인/닫기 전용)으로 문구 변경. 전체적으로 화살표 히트 영역 20×20/아이콘 13×13, 날짜 셀 높이 26px, 내부 여백도 줄여 캘린더 전체 크기를 이전보다 컴팩트하게 축소(이전엔 정사각형 셀이라 지나치게 커 보였음)
  • 시간 입력도 네이티브 <input type="time">에서 화면 2와 동일한 시:분 드롭다운(select)으로 교체해 두 화면의 "시간 설정" 디자인을 통일. 12시간제(오전/오후 선택 추가 필요) vs 24시간제를 비교한 결과, 24시간제가 입력 요소가 하나 적고(오전/오후 선택 불필요) 이미 화면2가 24시간제를 쓰고 있어 그대로 통일하기 좋다고 판단해 24시간제로 확정. 분 단위는 TIME_SLOT 생성 단위(30분)에 맞춰 00/30만 제공(화면2 안건마감 시간선택과 동일) — 15분 단위로 세분화하는 안도 검토했으나, 그러려면 이미 만들어진 화면1(슬롯 응답) 시간 그리드까지 셀을 2배로 재구성해야 해서 범위가 커져 이번엔 기존 30분 단위를 유지하기로 확정(문서·다른 화면 변경 없음)
  • 시간(시) 드롭다운 항목 수를 24개(0~23시)에서 12개(09~20시)로 축소 — 클릭 시 목록이 너무 길다는 피드백을 반영. 범위는 슬롯 응답 화면(화면 3, wireframe_01_slot_response_1.html)이 실제로 응답을 받는 시간대(9시~20시, for (let h = 9; h < 20; h++))와 동일하게 맞춤 — 회의실 현황판의 조회 범위도 09:00~20:00으로 통일
  • 검색 조건에 "인원" 필터 추가 — 날짜·시간 옆에 숫자 입력(인원)을 추가해, 값을 입력하고 "조회"를 누르면 해당 인원 이상 수용 가능한(capacityNum ≥ 입력값) 회의실만 리스트로 필터링해 보여줌. 인원을 비워두면 기존처럼 전체 회의실 표시. 조건에 맞는 회의실이 하나도 없으면 "조건에 맞는 회의실이 없어요" 안내로 대체. 입력 필드는 type="number"(스피너 버튼 있는 증감형)가 아니라 type="text" inputmode="numeric"로 순수 직접 입력만 받도록 구현 — 숫자가 아닌 문자는 입력 즉시 제거(휴대폰번호 입력과 동일한 필터링 방식)
  • 탭바 위치상 "예약" 탭 아이콘이 곧 이 화면의 진입점이라 뒤로가기 화살표 없음(화면 1·11·16과 동급)
