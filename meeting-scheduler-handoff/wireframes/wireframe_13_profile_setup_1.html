구현 노트 (Claude Code 참고용)

  • 기존 스펙엔 없던 완전 신규 화면 — 전통적인 "회원가입"을 대체함. 로그인(화면 13)에서 USER 레코드가 최초 생성될 때만(= 최초 로그인) 한 번 노출되고, 이후 재로그인 시에는 다시 노출되지 않음
  • 이름(USER.name)·이메일(USER.email)은 SSO/Google 인증 응답에서 이미 받아온 값이라 읽기 전용으로만 표시하고 이 화면에서 입력받지 않음
  • 부서·직위·직책(department, rank, position)도 이 화면에서 입력받지 않음 — 화면 2/3의 조직도에 이미 부서/직위/직책이 표시된다는 것은 회사 디렉토리(HRIS 또는 SSO 그룹/속성)가 이미 동기화돼 있다는 전제이므로, 최초 프로필 설정 시점에 디렉토리에서 자동으로 가져와 읽기 전용으로 저장함. 다만 이 화면(최초 설정)에서는 값을 화면에 노출하지 않고 안내 문구로만 짧게 언급 — 실제 값 확인은 "내 정보" 화면(신규)에서 하도록 함(이 화면은 최소 정보만 빠르게 입력받는 데 집중, 상단 계정 카드는 이름·이메일 2줄만 유지). 값이 틀려도 이 화면/앱에서 수정 불가 — 사내 HR/IT 시스템에서 갱신해야 함. 실제 구현 시 디렉토리 동기화 배치/API 연동 필요(db_model_mvp.md USER 테이블에 동기화 소스·최종 동기화 시각 필드 추가 고려)
  • 반면 phone(휴대폰), extension(내선번호), messenger_id는 회사 디렉토리에 없다고 가정하고 계속 수동 입력으로 유지 — 내선번호·메신저ID는 사내 메신저(Slack/Teams 등) 쪽 정보라 SSO/HRIS 표준 속성에 보통 없음, 휴대폰도 회사마다 동기화 여부가 달라 안전하게 수동 입력 유지
  • 필수 입력 최소화 — 휴대폰만 필수, 내선번호·메신저ID·프로필사진은 전부 선택 입력(데모에서는 휴대폰 입력 시 "시작하기" 버튼 활성화). 실제 필수 필드 정책은 재확인 필요
  • 프로필 사진(profile_image_url)은 SSO/Google 계정에 저장된 프로필 사진을 기본값으로 가져온다는 전제 — 그래서 "선택"이 아니라 아바타 위 작은 연필 아이콘(avatar-edit-btn)으로 "필요할 때만 변경"하는 형태로 설계함. 이미 기본값이 채워져 있다는 전제라 "나중에 하기"(건너뛰기) 개념은 제거함. 계정에 사진이 아예 없는 경우엔 기존 화면들과 동일하게 이름 첫 글자 이니셜 아바타로 대체 표시
  • "시작하기" 제출 시 확인 팝업 없이 바로 저장 후 온보딩(간단 사용법) 화면(신규)으로 이동 — 최초 사용자에게만 1회 노출되는 흐름이라 스킵 확인 팝업까지 넣으면 절차가 과해진다고 판단
  • 휴대폰번호·내선번호 입력 시 숫자만 입력해도 010-1234-5678 형태로 자동 하이픈 포맷팅 적용(formatPhoneNumber()). 와이어프레임은 자릿수 기반 단순 규칙(4자리 이하는 그대로, 이후 3-3-4/3-4-4 자릿수로 하이픈 삽입)이라 실제 구현 시에는 지역번호·내선 자리수 등 실제 규칙에 맞는 포맷 라이브러리(예: libphonenumber) 적용 및 커서 위치 처리 보강 필요
  • 이 와이어프레임은 단일 화면 목업이라 실제 이미지 업로드 대신 토스트로 대체 시연함
