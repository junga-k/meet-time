구현 노트 (Claude Code 참고용)

  • 기존 스펙엔 없던 완전 신규 화면 — 원래 검토 중이던 "로딩 스플래시"를 대체함. 최초 프로필 설정(화면 14) 완료 직후 딱 1회만 노출되고, 이후로는 다시 뜨지 않음(예: USER에 onboarding_seen_at 같은 플래그 추가 필요 — 현재 db_model_mvp.md엔 이 필드가 없어 스키마 반영 필요)
  • 4장 구성 — ①핵심 가치("자동으로 찾아드려요") ②필수→선택 단계적 응답수집 흐름 ③참석형태 사전선택+모드자동계산+회의실자동예약 ④마무리(알림 안내). 전부 실제 화면 흐름 그대로를 요약한 것이라, 화면 3·5·6·12 등의 실제 문구/로직이 바뀌면 이 온보딩 문구도 같이 갱신 필요
  • "건너뛰기" 링크와 마지막 슬라이드의 "시작하기" 버튼(4번째 슬라이드에서 "다음" 버튼 라벨이 "시작하기"로 바뀜) 둘 다 결과적으로 같은 곳(내 회의 목록, 화면 1)으로 이동
  • 페이지 이동 방식 — 점(dot) 인디케이터는 모양만으로는 클릭 가능해 보이지 않는다고 판단해 주된 상호작용에서 제외(눌러도 동작은 하지만 보너스 취급). 대신 (1) 4초마다 자동으로 다음 페이지로 넘어가는 자동재생(마지막 페이지에서는 멈춤), (2) 좌우 드래그/스와이프로 이전·다음 페이지 이동(Pointer Events 기반, 마우스 드래그와 터치 스와이프 모두 지원)을 기본 인터랙션으로 채택. 사용자가 "다음" 버튼을 누르거나 드래그/점 클릭 등 직접 조작을 하면 그 시점부터 자동재생은 멈춤(자동재생이 사용자가 읽고 있는 페이지를 강제로 넘기지 않도록)
  • 드래그로 이전 페이지로 돌아가는 것은 가능하지만 "다음" 버튼은 항상 다음 페이지로만 이동(뒤로가기 버튼 없음) — 다시 보고 싶은 페이지가 있으면 드래그로만 되돌아갈 수 있음
  • 이 와이어프레임은 단일 화면 목업이라 실제 페이지 전환 대신 토스트로 최종 이동을 표시함
