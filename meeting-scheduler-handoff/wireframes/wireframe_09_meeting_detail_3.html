구현 노트 (Claude Code 참고용)

  • 이 화면은 기본적으로 읽기 전용(확정된 회의의 정보 확인용)이며, 접근 대상은 MEETING.status='확정'인 회의의 전체 참석자. 유일한 예외가 아래 "참석으로 변경" 버튼 — 그 외 버튼(캘린더 추가, 회의록 이동)은 정보 열람·연동일 뿐 데이터를 바꾸지 않음
  • 상단 [데모 전용] 컨트롤은 이 화면을 보는 로그인 사용자(여기서는 선택 참석자 "정다은")의 confirmation_status와 회의 종료 여부를 바꿔가며 세 가지 경우를 한 파일에서 확인하기 위한 와이어프레임 전용 장치이며 실제 화면에는 없음
  • "참석으로 변경" 버튼(신규 — 이번 요청으로 추가): 선택 참석자가 압축 후보 전부 불가(화면 12) 또는 참석재확인 불참 통보(화면 8)로 confirmation_status='불참'이 된 뒤, 최종 확정 시점에 발송되는 NOTIFICATION(type: 불참안내)을 보고 이 화면으로 들어왔을 때 노출됨. 노출 조건은 본인이 선택 참석자 && confirmation_status='불참' && 아직 회의 시작 전 세 가지 모두. 클릭 시 확인 팝업("참석으로 변경하시겠어요?") → 확인하면 confirmation_status만 '확정'으로 되돌리고 재조율은 트리거하지 않음(이미 필수 참석자 기준으로 확정된 시간에 합류하는 것뿐이라 시간 자체는 그대로). 상세 배경은 screen_specs.md 화면 12 "선택 참석자 불참 처리 후속 안내" 절, db_model_mvp.md NOTIFICATION.type의 불참안내 항목 참고
  • 회의록 접근 규칙 갱신(§2-49 재조정) — "회의 종료 후"가 아니라 "회의 시작 후"부터 접근 가능으로 최종 확정. 회의가 시작된 순간(진행 중 또는 종료, meetingStarted())부터는 확정/불참 상태와 무관하게 하단 버튼이 "회의록 확인하기" 하나로 통일되어 화면10(회의록·향후 추진 과제, `wireframe_17_meeting_minutes_1.html`)으로 연결된다(데모라 토스트로 이동 시뮬레이션). 회의 시작 전에는 기존과 동일하게 확정 참석자는 "캘린더에 추가" + 비활성 "회의록·향후 추진 과제 이동 (회의 시작 후)", 불참 참석자는 "참석으로 변경" + "캘린더에 추가"가 노출된다. "참석으로 변경" 버튼도 동일하게 회의가 시작되는 순간 사라짐(기존엔 "종료" 기준이었으나 시작 시점 기준 필수 조건과 맞춰 통일).
  • 데모 컨트롤을 2개 축으로 분리(참석 상태: 확정/불참 × 회의 진행 상태: 회의 전/진행 중/회의 종료) — 기존엔 3개 시나리오 칩 하나로 묶여있어 "확정 참석자 + 회의 종료" 같은 조합을 보여줄 수 없었는데, 이번에 두 축을 독립적으로 토글할 수 있게 바꿔 모든 조합(확정×회의전/진행중/종료, 불참×회의전/진행중/종료)을 한 파일에서 확인 가능
  • 장소/화상링크: ROOM_BOOKING → ROOM.name/ROOM.location과 MEETING.video_link를 함께 표시(하이브리드 모드 가정, wireframe_06_dashboard.html과 동일한 회의실·화상링크 데모 데이터를 재사용해 화면 간 일관성 유지)
  • 참석자 목록: 필수/선택 역할별로 묶어 표시하고, 각자의 confirmation_status(확정/불참)와 슬롯 응답 시 선택했던 attendance_mode(대면/온라인)를 함께 보여줌. "나"에 해당하는 행은 배경을 옅게 강조하고 "나" 태그를 붙여 구분
  • 안건 목록은 AGENDA.title과 작성자, is_required(필독) 여부만 요약 표시 — 상세 내용·첨부 확인은 화면 4(안건 등록/수정 화면)에서 이어지는 것으로 가정(이 화면에서는 목록 열람까지만, 별도 상세 진입 화면은 스펙에 없어 이 와이어프레임에도 추가하지 않음)
  • 이 화면과 wireframe_06_dashboard.html의 회의 제목·확정 시간·회의실·참석자 이름은 동일한 가상의 회의("3분기 로드맵 논의")를 공유하도록 맞췄지만, 각 와이어프레임 파일은 독립된 데모 데이터를 갖는 기존 관례상 세부 수치(참석형태 배분 등)까지 완벽히 일치시키진 않음

"회의 종료됨"도 같은 이유로 "회의 종료"로 축약. 확정 상태에는 대칭으로 "참석" 배지(초록)를 표시
