## 2. 이미 확정된 신규 결정사항 (문서 역반영 대상)

2. `NOTIFICATION.type`에 `응답요청`, `불참안내` 2종 추가 필요 (아직 `db_model_mvp.md`에 실제 반영 안 됨 — §3-25 참고)
3. 화면 5에 주최자용 "리마인드" 버튼 신규
4. 사용자 노출 문구 순화: "완화요청"→"변경 요청", "거절"→"불가" (화면 6·7·11). 시스템/DB 용어는 "완화요청" 그대로 유지
5. 화면 8 "온라인 전환" 버튼 조건부 분기 (원래 참석형태로 가능하면 "참석 확정"만, 어려워진 경우만 "참석 확정·온라인 전환")
6. 참석형태 미선택 시 그리드/후보 잠금 (화면 3·12)
7. 화면 6에 "확정된 회의 시간 배너" 신규 — `screen_specs.md` 화면 5 표시 데이터 목록에 원래 빠져있던 항목 보완
8. 탭바 높이 56px로 통일 (스펙 문서 85px → 56px 값만 수정하면 됨)
9. **부서·직급·직책은 회사 디렉토리(HRIS/SSO 그룹 속성) 자동 동기화 + 읽기 전용**으로 확정 — 최초 프로필 설정(화면13) 화면에서 더 이상 수동 입력받지 않음. 단, 이 화면에는 실제 값을 노출하지 않고(화면이 복잡해 보이는 것을 피하기 위해) 안내 문구로만 짧게 언급 — 실제 값은 "내 정보" 화면(화면16)에서 확인. 반면 휴대폰·내선번호·메신저ID는 회사 디렉토리에 없다고 가정하고 계속 수동 입력 유지. `db_model_mvp.md` USER 테이블에 동기화 소스/최종 동기화 시각 필드 추가 여부 검토 필요
10. 최초 프로필 설정 화면의 프로필 사진은 SSO/Google 계정 사진을 기본값으로 가져온다는 전제로 변경 — "사진 선택"(선택/스킵 UI) 대신 아바타 위 작은 편집 아이콘으로 축소, "나중에 하기" 제거
11. 최초 프로필 설정 화면 상단의 "최초 로그인 · 1/1" 스텝 표시 및 "회사 계정에 없는 몇 가지 정보만 채우면 바로 시작할 수 있어요" 안내문구 제거 — 화면을 더 간결하게
12. 휴대폰번호·내선번호 입력 시 숫자만 입력해도 자동으로 하이픈 포맷(예: 010-1234-5678) 적용 — 실제 구현 시 정식 포맷 라이브러리 적용 필요
13. 폼 필드에 placeholder만 쓰지 않고 별도 라벨(예: "휴대폰번호(필수)") 표시하는 방식으로 통일 — 입력 중 라벨이 사라지지 않도록
14. 폼 화면 텍스트 색상 위계 확정 — **섹션 제목**(예: "연락처", "프로필 사진")은 진한 색(ink), **필드 라벨**(예: "휴대폰번호")은 옅은 색(muted)으로 역전 배치. 다른 폼 화면에도 동일 규칙 적용 검토 필요
15. 폼 화면 간격 규칙 확정 — 섹션(계정/프로필사진/연락처) 간 32px, 섹션 내 제목-콘텐츠 간 16px, 필드 그룹(라벨+입력창) 간 12px(그룹 내부 라벨-입력창은 밀착). 상태바와 화면 제목 사이도 32px. 다른 폼 화면에도 동일 간격 규칙 적용 검토 필요
16. **하단 CTA 버튼 영역(`.footer`)의 굵은 구분선(`border-top: 2px solid var(--ink)`) 제거를 디자인 시스템 규칙으로 확정, 단 스크롤 있는 화면은 "스크롤 중일 때만 동적 그림자" 방식 적용** — 스크롤이 없는 짧은 화면(화면13 프로필설정, 화면14 온보딩)은 배경색 차이만으로 구분해 선/그림자 완전히 없앰. 반면 스크롤이 발생하는 화면(화면03 제출바·조직도 모달, 05, 06, 07, 09, 10, 15)은 콘텐츠가 더 남아있을 때만 버튼 영역 위에 옅은 그림자(`box-shadow`)가 나타나고, 맨 아래까지 스크롤하면 그림자가 사라져 스크롤 없는 화면과 동일한 모습이 되도록 JS로 구현(`show-boundary` 클래스, 스크롤 컨테이너의 `scroll` 이벤트로 토글). 앞으로 만들 화면에도 이 규칙(스크롤 있으면 동적 그림자, 없으면 선 없음) 적용
